Overview

Why I built this project

This project was created as the foundation of my online presence. I wanted a portfolio that does more than list projects: it should reflect how I think about structure, visual hierarchy, performance, and user experience. Because this is my first showcased project, it also works as a statement of how I approach web development.

Goals

What the site should communicate

  • A clear introduction to who I am and what I focus on
  • Projects presented with structure instead of visual clutter
  • A responsive layout that feels clean on desktop and mobile
  • A design direction inspired by gaming and modern interface systems

Version Control (Git)

How I use Git in this project

  • Track progress with commits instead of creating manual folder backups
  • Use staging to separate finished work from half-finished changes
  • Keep local-only files like .vs out of the repository with .gitignore
  • Understand the basics of branches, fetch, push, pull, and pull requests

Localization

How I structured multilingual content

  • Separated translation content into a reusable translations file
